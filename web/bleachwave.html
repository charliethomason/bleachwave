Summary: Bleachwave originated as "Bleachwave Productions," an art and creative writing project I started with a friend of mine. The project was retired in Fall 2011 and then, when I graduated from Code Academy in March 2012, I resurrected the name for an online printmaking store I was building. Eventually the concept of the site grew and evolved into a multi-faceted web development portfolio, art gallery, personal bio/resume, blog, and photo blog. The site, now called "Bleachwave: The Website of Charlie Thomason," would be the all-encompassing home on the Internet for everything I do and sell, as well as a tool for me to pursue new employment opportunities. For the online print store aspect of the site, I decided to continue using 3rd party print-on-demand services such as Society6. The first version of the site was launched the day after my 28th birthday (June 18, 2012).

Bleachwave was originally going to be built with Ruby on Rails; I later decided that this would only over-complicate things for me. Since the site would not initially have any users or transactions occurring, there was no reason why it needed to be built on a database. For at least the portfolio and informational parts of the site, I built the site entirely with HTML5, CSS3, and JQuery. I created a responsive layout for the site that would be better optimized for mobile browsers, and implemented the Elastislide JQuery script to allow touchscreen users to easily swipe through the Web Portfolio and Art Gallery carousels (on the home page and all portfolio pages).

Many of the websites featured in the Web Portfolio were, sadly, no longer on the Web. Therefore, I had to create a sub-domain of Bleachwave.com where I could house modified re-creations of the original sites. I also created a robots.txt file for the sub-domain directory to make sure Google and other search bots would not try to re-index those sites.

For the Photo Blog, I wanted to create a singular home on the Web for all of my photography. As I had recently abandoned Flickr, I decided a Tumblr blog would be the most convenient option. I started with "The Minimalist" theme and linked the Tumblr blog to a custom stylesheet on my own server to make it look identical to the rest of the site. I also linked my Instagram account to the Tumblr blog so that it could serve as a home for both my mobile and dSLR photography.
